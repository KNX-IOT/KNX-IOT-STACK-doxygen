◆ MY_NAME

#define MY_NAME   "Actuator (LSAB) 417"

The name of the application.

Application Design

support functions:

  • app_init initializes the stack values.
  • register_resources function that registers all endpoints, e.g. sets the GET/PUT/POST/DELETE handlers for each end point

  • main starts the stack, with the registered resources. can be compiled out with NO_MAIN

    handlers for the implemented methods (get/post):

    • get_[path] function that is being called when a GET is called on [path] set the global variables in the output
    • post_[path] function that is being called when a POST is called on [path] checks the input data if input data is correct updates the global variables

stack specific defines

  • linux build for linux
  • WIN32 build for windows
  • OC_OSCORE oscore is enabled as compile flag

    File specific defines

  • NO_MAIN compile out the function main()
  • INCLUDE_EXTERNAL includes header file "external_header.h", so that other tools/dependencies can be included without changing this code

register all the resources to the stack this function registers all application level resources:

  • each resource is

    • observable
    • discoverable
    • used interfaces, including the default interface. default interface is the first of the list of interfaces as specified in the input file
